- Keep the commit message short: a single line, ideally under 50 characters, no more than 72.
- Use the imperative mood (e.g. "fix bug" not "fixed bug" or "fixes bug").
- Summarize only the core change — no explanations, no bullet lists, no body text.
- Do not include ticket numbers, file names, or verbose descriptions unless critical.
- Format: `<type>: <short summary>` (e.g. `fix: null pointer on login`, `feat: add dark mode toggle`).
- No period at the end of the summary line.
